# 컨테이너 가상화

> "가상화"란 컴퓨터 리소스를 추상화해 여러 환경을 하나의 하드웨어 위에서 독립적으로 실행하는 기술이다.

## 가상화 3종류 비교

| 종류 | 구조 | 특징 |
|------|------|------|
| **호스트 가상화** | 호스트 OS → 하이퍼바이저 → 게스트 OS × N | 구성이 쉽지만 오버헤드 큼 |
| **하이퍼바이저 가상화** | 하이퍼바이저 → 게스트 OS × N (호스트 OS 없음) | 성능 우수, 초기 설정·관리 어려움 |
| **컨테이너 가상화** | 호스트 OS → 도커 엔진 → 컨테이너 × N | 가장 가볍고 빠름 — OS를 공유하기 때문 |

### 왜 컨테이너가 VM보다 가볍나?

VM은 게스트 운영체제 전체를 포함한다. 컨테이너는 호스트 OS 커널을 **공유**하고 프로세스 수준에서만 격리한다. 격리 메커니즘이 바로 **네임스페이스**.

## 네임스페이스 (Namespace)

- 프로세스 내에서 시스템 리소스를 **분리**하기 위한 리눅스 커널 기능
- 컨테이너가 서로를 "다른 세상"처럼 보게 만드는 근거
- 네임스페이스 종류: PID, Network, Mount, UTS, IPC, User 등
- → 컨테이너는 결국 "네임스페이스로 격리된 프로세스 묶음"

## OS 기초 용어 (Docker 이해를 위한 최소 지식)

| 용어 | 설명 |
|------|------|
| 프로그램 | 실행 가능한 명령어 집합 — 디스크에 있고 메모리에 아직 안 올라감 |
| 프로세스 | 실행 중인 프로그램 (메모리에 적재된 상태) |
| 멀티 프로세스 | 같은 프로그램을 여러 번 실행해 여러 프로세스를 띄우는 것 |
| 스레드 | 프로세스 내부 실행 흐름의 단위 |
| 셸 | OS와 상호작용하는 매개체 (bash, zsh 등) |
