# 마크다운 (Markdown)

## 한 줄 정의

일반 텍스트에 간단한 기호로 서식을 입히는 **경량 마크업 언어**. 파일 확장자 `.md`.

## 왜 쓰는가

- **플레인 텍스트**라 어떤 편집기·OS에서도 열리고, 특정 프로그램에 묶이지 않는다.
- 한 줄 변경이 그대로 드러나 **버전 관리(git)**와 잘 맞는다.
- ChatGPT 같은 AI 도구와 코딩 환경의 **공통 형식** — 한 번 익히면 두루 쓰인다.
- [[obsidian]]을 비롯한 많은 노트 앱이 마크다운을 저장 형식으로 쓴다.

## 기본 문법

| 요소 | 문법 |
|------|------|
| 헤딩 | `# 제목`, `## 소제목` |
| 목록 | `- 항목` (글머리), `1. 항목` (번호) |
| 강조 | `**굵게**`, `*기울임*` |
| 코드 | `` `인라인` ``, 삼중 백틱 코드블록 |
| 인용 | `> 인용문` |
| 링크 | `[텍스트](url)` |

## frontmatter

문서 맨 위 `---` 사이에 적는 YAML 메타데이터. 마크다운 본문은 아니지만
도구가 읽어 검색·분류에 쓴다 → [[obsidian-properties]].

## NoteWiki와의 관계

이 세컨드 브레인의 **모든 파일이 마크다운**이다 — `raw/`·`wiki/`·`Output/`의 노트,
`CLAUDE.md`, 이 페이지까지. AI가 읽고 쓰기 쉬운 형식이라 위키 전체의 토대가 된다.
