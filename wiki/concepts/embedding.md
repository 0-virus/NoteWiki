# Embedding (임베딩)

## 한 줄 정의

텍스트(또는 이미지·오디오)의 **의미**를 고정 길이의 **숫자 벡터**로 변환하는 것. 의미가 비슷할수록 벡터 공간에서 가까이 위치한다.

## 왜 이게 가능한가 — 원리

임베딩 모델은 거대한 양의 텍스트로 사전 학습되어, "비슷한 맥락에서 등장하는 단어·문장은 비슷한 벡터로 표현한다"는 분포 가설(distributional hypothesis)을 학습한다.

예시 (개념적):
```
"강아지" → [0.21, -0.43, 0.88, ..., 0.12]   (1536차원)
"개"     → [0.20, -0.41, 0.89, ..., 0.10]   ← "강아지"와 매우 가까움
"자동차" → [-0.55, 0.71, -0.12, ..., 0.66]  ← 멀리 떨어짐
```

이 좌표계 위에서 **"가깝다 = 의미가 비슷하다"**가 성립한다. 그래서 검색이 키워드 일치가 아닌 **의미 유사도**로 가능해진다.

## 거리 측정 — 코사인 유사도

벡터 간 "가까움"은 보통 코사인 유사도(cosine similarity)로 잰다:

$$
\text{cos}(A, B) = \frac{A \cdot B}{\|A\| \|B\|}
$$

- 결과 범위: -1 ~ 1
- 1에 가까울수록 의미가 같음, 0이면 무관, -1이면 정반대
- 벡터 길이(magnitude)는 무시하고 **방향**만 본다 → 문서 길이에 영향받지 않음

## ⚠️ 주의 — 같은 모델로 쿼리하라

[[rag]]에서 가장 흔한 실수: **인덱싱 때 쓴 임베딩 모델과 쿼리 때 쓰는 모델이 달라지면 같은 좌표계가 아니다.** 두 벡터를 비교해도 의미가 없다.

→ 모델 버전을 갈아끼우려면 **전체 재인덱싱** 필요.
