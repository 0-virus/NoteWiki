# CSS 명시도와 우선순위

## 한 줄 정의

한 요소에 여러 CSS 규칙이 충돌할 때, 브라우저가 **어떤 규칙을 적용할지 고르는 순위 체계**.

## 왜 필요한가

CSS는 여러 곳(라이브러리, 전역 스타일, 컴포넌트)에서 같은 요소를 동시에 노린다.
브라우저는 그중 하나의 값을 골라야 하므로, 결정 규칙이 필요하다.

## 결정 순서

규칙은 **위에서 아래 순서**로 비교된다. 앞 단계에서 승부가 나면 뒤는 보지 않는다.

1. **중요도**
   `!important` > 인라인 스타일 > 일반 CSS > 브라우저 기본 스타일
   - `!important`는 가장 강하지만 남발 금지 — 우선순위 추적이 불가능해진다.
2. **명시도(Specificity)** — [[css-selector|선택자]]가 얼마나 구체적인지
   인라인 > ID(`#id`) > 클래스·속성·가상클래스(`.cls`, `[type]`, `:hover`) > 태그·가상요소(`div`, `::before`) > 전체선택자(`*`)
3. **작성 순서** — 명시도가 같으면 **나중에 쓴 규칙**이 이긴다.

## 실무 적용

- 충돌을 피하려면 선택자를 구체적으로: `.header .nav-item`, `.button.primary`.
- **라이브러리 CSS를 먼저 import**, 커스텀 CSS를 나중에 import해야 내 스타일이 라이브러리를 덮어쓴다.
- 개발자 도구(F12)에서 우선순위에 밀린 스타일은 취소선으로 표시된다.
