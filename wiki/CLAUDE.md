# wiki/ — 위키 (AI가 컴파일하는 지식 베이스)

> 이 폴더는 **AI가 전적으로 소유·유지**한다. AI가 페이지를 만들고, 갱신하고, 연결한다.
> 영균은 읽고, AI는 쓴다. 위키는 원본을 추가할 때마다 더 풍부해지는 **누적 자산**이다.

## 폴더 구조

| 폴더/파일   | 역할                                                                                |
| ----------- | ----------------------------------------------------------------------------------- |
| `concepts/` | 개념 페이지. 하나의 개념 = 하나의 파일 (예: `servlet.md`)                           |
| `flows/`    | 기술 활용 흐름 페이지 (예: `tomcat-to-spring-mvc.md`) — 박영균이 가장 원하는 "연결" |
| `sources/`  | 원본 소스별 요약 페이지. `raw/`의 각 원본에 대응                                    |
| `index.md`  | 위키 전체 카탈로그. 모든 페이지의 링크 + 한 줄 요약                                 |
| `log.md`    | 시간순 작업 기록 (append-only)                                                      |

## 페이지 작성 규칙

- 파일명: 소문자 kebab-case (예: `spring-mvc.md`).
- 상단에 YAML frontmatter:
  ```yaml
  ---
  type: concept | flow | source
  tags: [tomcat, was]
  updated: 2026-05-18
  sources: [raw/articles/xxx.md]
  ---
  ```
- 본문은 주제별 **자유 형식**. 단, 관련 페이지를 `[[page-name]]` **위키링크**로 연결한다. 일반 태그는 frontmatter `tags`에 둔다.
- 새 정보가 기존 서술과 **모순되면** 덮어쓰지 말고 `> ⚠️ 모순:` 으로 명시한다.
- 새 지식은 기존 페이지를 대체하지 않고 **새 가지(섹션/페이지)로 추가**한다. 단, 명백한 오류·오타·낡은 설명은 수정하고 `log.md`에 남긴다.
- 새 위키링크를 만들 때는 대상 페이지가 실제로 존재해야 한다. 아직 만들지 않을 개념은 일반 텍스트나 `TODO:`로 남기고, 의도적으로 보류한 이유를 적는다.

## 작업 시 (루트 CLAUDE.md의 3대 작업 참조)

- **Ingest / Query 후** 반드시 `index.md`와 `log.md`를 갱신한다.
- 작업 후 새 위키링크의 대상 파일이 존재하는지 점검한다.
- **Query**의 가치 있는 답변은 위키에 새 페이지로 환류한다.
