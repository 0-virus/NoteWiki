# Proxy(프록시)란? Forward vs Reverse Proxy — 코딩문

> **원본**: [YouTube · 코딩문 채널, 2024-06-03, 약 7분](https://www.youtube.com/watch?v=Rt-KdCpsmdc)
> **클리핑**: 2026-05-21

## 맥락 (왜 저장했나)

영균이 **부트캠프 팀 프로젝트** 진행 중 **다른 팀의 발표·설명**에서
"리버스 프록시"·"로드 밸런서" 용어를 접했지만 정확히 이해하지 못한 상태였다.
이번 기회에 **시스템 디자인의 기초 인프라 어휘**를 자기 위키에 못 박아두려고 캡처.
바로 이어지는 [[load-balancing-video]]와 묶어 한 가지로 다룬다.

## 핵심 주장 (Key Claims)

1. **프록시 = 네트워크 통신의 중간자**. 두 엔티티(클라이언트↔서버 또는 서버↔서버)
   사이를 중계하면서 상호작용을 **감추거나 제어**한다.
2. **Forward Proxy는 클라이언트 쪽에 위치**한다 → 클라이언트를 보호한다.
   - 클라이언트 IP를 외부에 숨김 (서버는 프록시 IP만 봄)
   - 특정 IP/도메인 접근 제어 (콘텐츠 필터링)
   - 캐싱으로 응답 지연 감소
   - 외부 트래픽 모니터링·로깅
   - 사례: 학교/기업/공공기관의 사내망 게이트웨이
3. **Reverse Proxy는 서버 쪽에 위치**한다 → 서버를 보호한다.
   - 실제 서버 IP를 클라이언트가 알 수 없음 → DDoS 등 직접 공격 난이도↑
   - 캐싱 레이어 / 로드 밸런서 역할
   - 대표 구현: **Nginx**
4. **두 프록시는 "어느 쪽에 붙느냐"가 본질**이다. 동작 원리는 같은 중계자지만
   **누구를 익명화하느냐**가 정반대다.

## 자주 헷갈리는 포인트 (이 영상이 정리해준 것)

- "프록시가 한 종류인 줄 알았다" → **Forward / Reverse는 위치만 다른 한 쌍**임을 못 박는다.
- "Nginx = 웹 서버" 라고만 알고 있었다 → 실제로는 **리버스 프록시 + LB가 본업**이고
  정적 파일 서빙은 부가 기능이다.
- IP 익명화는 보안 기능이지만, **"누구의 IP를 숨기느냐"** 가 Forward와 Reverse를 가른다.
