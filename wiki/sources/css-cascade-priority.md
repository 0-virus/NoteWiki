# 소스: CSS 참조 우선순위

> 원본: `raw/notes/CSS 참조 우선순위.md` — 직접 작성 필기(노션 이전분).

## 맥락 (왜 기록했나)

기업 해커톤에서 CSS 우선순위 문제로 고생한 뒤, 같은 문제가 또 발생해 정리한 노트.
실전에서 막힌 지점을 원리로 해소한 기록.

## 핵심 요약

- CSS 우선순위는 **중요도 → 명시도(Specificity) → 작성 순서** 순으로 결정된다.
- 중요도: `!important` > 인라인 스타일 > 일반 CSS > 브라우저 기본.
- 명시도: 인라인 > ID > 클래스/속성/가상클래스 > 태그/가상요소 > 전체선택자(`*`).
- 명시도가 같으면 **나중에 작성한 규칙**이 적용된다.
- 실무 팁: 라이브러리 CSS를 먼저 `import`, 커스텀 CSS를 나중에 `import`해야 덮어쓸 수 있다.
- CSS Modules(`.module.css`)는 클래스명을 `파일명_클래스명__해시`로 변환해 전역 충돌을 막는다.
