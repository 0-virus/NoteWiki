# 소스: 제네릭, 람다, 함수형 인터페이스, 익명 클래스

> 원본: `raw/notes/제네릭, 람다, 함수형 인터페이스, 익명 클래스.md` — 직접 작성 필기(노션 이전분).
> 네 개념을 따로가 아니라 **하나의 조합**으로 엮어 정리한 노트.

## 맥락 (왜 기록했나)

부트캠프 Java 학습 중 정리한 **실무 대비 자산**. "함수를 값처럼 다루는" Java의
네 장치가 어떻게 맞물리는지를 `IAdd<T>` 예제 하나로 관통하며 이해한 기록.

## 핵심 요약

- **제네릭**: 타입을 파라미터처럼 받는 기능(`Box<T>`). 컴파일 타임 타입 체크 +
  캐스팅 제거. 참조 타입만 가능.
- **함수형 인터페이스**: 추상 메서드 1개짜리 인터페이스. 람다의 타겟 타입.
- **람다**: 함수형 인터페이스의 유일한 메서드 구현을 짧게 쓴 문법. 실제로는 객체 생성.
- **익명 클래스 vs 람다**:
  - `this` — 익명 클래스는 자기 자신, 람다는 바깥 객체.
  - 스코프 — 익명 클래스는 독립(필드 선언 가능), 람다는 바깥 메서드 지역변수 공유.
  - 적용 — 익명 클래스는 메서드 여러 개·추상 클래스도, 람다는 함수형 인터페이스만.
- 제네릭이 람다의 매개변수·리턴 타입을 자동 확정 (`IAdd<Integer>` → `Integer add(...)`).
