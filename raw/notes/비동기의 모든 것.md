# JavaScript 비동기 프로그래밍

---

## 1. 동기 vs 비동기

### 동기 (Synchronous)

작업이 순서대로 실행되고, 앞 작업이 끝나야 다음으로 넘어가는 방식.

```
작업 1 완료 → 작업 2 완료 → 작업 3 완료
```

- 코드가 직관적이고 예측하기 쉬움
- 오래 걸리는 작업이 있으면 **그 동안 아무것도 못 함 (블로킹)**

### 비동기 (Asynchronous)

작업을 시작해놓고, 결과를 기다리는 동안 다른 일을 할 수 있는 방식.

```
작업 1 시작 → (기다리는 동안 다른 작업 수행) → 작업 1 완료 신호 → 이후 처리
```

- JavaScript는 **싱글 스레드** 언어이기 때문에 비동기가 특히 중요함
- 네트워크 요청, 파일 읽기 같은 I/O 작업을 동기로 처리하면 그동안 UI가 완전히 멈춤

## 2. JavaScript의 비동기 구현 방식

### ① Callback

가장 오래된 방식. 함수가 완료됐을 때 호출할 함수를 인자로 넘김.

```jsx
fetchData(url, function (result) {
  process(result, function (processed) {
    save(processed, function () {
      console.log("완료");
    });
  });
});
```

**콜백 지옥 (Callback Hell)**

중첩이 깊어질수록 코드가 오른쪽으로 계속 밀려나는 현상. "Pyramid of Doom"이라고도 불림.

- 완료 후 할 일을 함수 안에 직접 중첩해야 함
- 단계가 늘어날수록 가독성이 급격히 떨어짐
- 각 단계마다 에러 처리까지 하면 더욱 복잡해짐

---

### ② Promise

ES6에 등장. 비동기 작업의 성공/실패를 나타내는 객체.

**3가지 상태**

| 상태        | 설명                         |
| ----------- | ---------------------------- |
| `pending`   | 아직 완료되지 않은 초기 상태 |
| `fulfilled` | 작업 성공 → `.then()` 실행   |
| `rejected`  | 작업 실패 → `.catch()` 실행  |

**특성: 상태 불변성**

한 번 상태가 바뀌면 다시는 바뀌지 않음. `resolve()`나 `reject()`를 두 번 호출해도 첫 번째만 유효.

```jsx
new Promise((resolve, reject) => {
  resolve("첫 번째"); // 이게 적용됨
  resolve("두 번째"); // 무시됨
  reject("에러"); // 무시됨
});
```

**기본 사용법**

```jsx
fetch("/api/data")
  .then((res) => res.json())
  .then((data) => console.log(data))
  .catch((err) => console.error(err));
```

콜백과의 차이: 완료 신호를 객체로 들고 다니면서 `.then()`으로 밖에서 자유롭게 연결 가능.

---

### resolve() / reject() 동작 원리

`new Promise()`에 넘기는 함수는 JS 엔진이 `resolve`와 `reject`를 자동으로 만들어서 인자로 넘겨줌.

```jsx
new Promise((resolve, reject) => {
  setTimeout(() => {
    resolve("완료!"); // 1초 후 상태 변경 + 값 전달
  }, 1000);
});
```

- `resolve(값)` 호출 → `pending → fulfilled`, `.then()` 또는 `await` 이후 코드에 값 전달
- `reject(에러)` 호출 → `pending → rejected`, `.catch()` 또는 `try/catch`로 에러 전달

### Promise 정적 메서드

여러 개의 Promise를 한꺼번에 다룰 때 사용.

| 메서드               | 동작                              | 특징                                     |
| -------------------- | --------------------------------- | ---------------------------------------- |
| `Promise.all`        | 모두 성공하면 결과 배열 반환      | 하나라도 실패하면 즉시 reject            |
| `Promise.allSettled` | 성공/실패 관계없이 모두 완료 대기 | 절대 reject 안 됨, 각 결과에 status 포함 |
| `Promise.race`       | 가장 먼저 완료된 것의 결과 반환   | 성공이든 실패든 상관없이 가장 빠른 것    |
| `Promise.any`        | 가장 먼저 **성공한** 것 반환      | 실패는 무시, 모두 실패해야 reject        |

> `race` vs `any` 구분: race는 실패도 결과로 인정, any는 성공만 결과로 인정

**`Promise.race` 타임아웃 구현 예시**

```jsx
const result = await Promise.race([
  fetch("/api/data"),
  new Promise((_, reject) => setTimeout(() => reject("timeout"), 3000)),
]);
```

### ③ async / await

ES2017에 등장. Promise를 더 읽기 쉽게 만든 **문법 설탕 (syntactic sugar)**.

내부적으로는 Promise와 동일하게 동작하지만, 동기 코드처럼 읽힘.

```jsx
async function loadUser() {
  try {
    const res = await fetch("/api/user"); // 결과 나올 때까지 일시정지
    const data = await res.json(); // 동기처럼 읽히지만 비동기
    return data;
  } catch (err) {
    console.error(err);
  }
}
```

**핵심 규칙**

- `await`는 Promise를 반환하는 것 앞에만 의미가 있음
- `async` 함수는 내부에서 뭘 return하든 항상 Promise로 감싸서 반환함
- `await`는 Promise가 아닌 값에도 쓸 수 있지만, 즉시 반환되므로 의미 없음

```jsx
async function getNumber() {
  return 42; // 실제로는 Promise.resolve(42)가 반환됨
}

const result = await getNumber(); // 그래서 await 가능
console.log(result); // 42
```

## 3. 세 가지 방식 비교

같은 동작을 세 가지 방식으로 표현:

```jsx
// ① Callback
setTimeout(() => console.log("1초 후"), 1000);

// ② Promise
new Promise((resolve) => setTimeout(resolve, 1000)).then(() =>
  console.log("1초 후"),
);

// ③ async/await
async function wait() {
  await new Promise((resolve) => setTimeout(resolve, 1000));
  console.log("1초 후");
}
```

콜백 지옥 vs async/await 비교 (유저 → 포스트 → 댓글 순서로 fetch):

```jsx
// ❌ Callback 방식 — 콜백 지옥
fetchUser(userId, (user) => {
  fetchPosts(user.id, (posts) => {
    fetchComments(posts[0].id, (comments) => {
      console.log(comments);
    });
  });
});

// ✅ async/await 방식
const user = await fetchUser(userId);
const posts = await fetchPosts(user.id);
const comments = await fetchComments(posts[0].id);
console.log(comments);
```

**발전 흐름 요약**

```
Callback → Promise → async/await
  (구식)    (개선)     (현재 표준)
```

- Callback ⊂ Promise ⊂ async/await 구조로 발전
- async/await을 쓰더라도 Promise를 이해해야 하고, Promise를 이해하려면 Callback이 뭔지 알아야 함
