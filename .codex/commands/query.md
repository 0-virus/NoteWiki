# Codex Query

`wiki/` 문서를 근거로 질문에 답하는 절차다. 벡터 검색이 아니라 `wiki/index.md` 기반 탐색을 쓴다.

## 절차

1. 질문이 비어 있으면 무엇을 묻고 싶은지 먼저 확인한다.
2. `wiki/index.md`를 읽어 관련 페이지 후보를 찾는다.
3. 후보 `concepts/`, `flows/`, `sources/` 페이지를 읽는다.
4. 필요한 경우 페이지 안의 위키링크를 따라가거나 `rg`로 `wiki/`를 보조 검색한다.
5. 읽은 내용을 종합해 한국어로 답한다.
6. 근거가 된 페이지를 `[[page-name]]` 형태로 표시한다.
7. 위키에 없는 내용은 추측과 구분해서 말한다. 최신 정보가 필요한 경우에는 확인 필요성을 명시한다.
8. 답변이 비교·분석·새 연결로 재사용 가치가 크면 `wiki/`에 환류한다.
9. 환류한 경우 `wiki/index.md`를 갱신하고, 새 위키링크 대상 존재 여부를 점검한다.
10. `wiki/log.md`에 `## [YYYY-MM-DD] query | <질문 요약>` 항목을 append한다.

## 답변 기준

- 영균의 학습 단계와 실무 적용 가능성을 함께 고려한다.
- "왜 이렇게 동작하는가"를 생략하지 않는다.
- 표면적 요약보다 개념 간 연결을 보여준다.
- 출처 없는 일반 지식은 위키 근거와 분리한다.
