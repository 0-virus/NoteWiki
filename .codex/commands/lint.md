# Codex Lint

NoteWiki 전체 건강 점검 절차다. `raw/`는 수정하지 않고, 필요한 경우 `wiki/`와 지시 파일만 정리한다.

## 점검 항목

- 깨진 위키링크: `[[page-name]]` 대상 파일이 없는 경우
- 고아 페이지: 들어오는 링크가 없는 페이지
- index 불일치: `wiki/index.md` 목록·통계와 실제 파일 수가 다른 경우
- log 불일치: `wiki/log.md`에 기록된 작업이 실제 파일에 반영되지 않은 경우
- 누락 개념: 여러 페이지에서 언급되지만 독립 페이지가 없는 중요 개념
- 누락 상호참조: 관련 있는데 서로 링크되지 않은 페이지
- 낡은 내용: 최신 소스·프로젝트 맥락과 어긋난 설명
- frontmatter 누락 또는 `updated`, `sources` 불일치

## 수정 기준

- 명백한 오류·오타·낡은 설명은 수정한다.
- 판단이 필요한 새 개념 신설은 사용자에게 제안하거나, 작고 명확하면 생성한다.
- 새 위키링크를 만들면 대상 페이지가 존재하는지 확인한다.
- 아직 만들지 않을 개념은 위키링크로 만들지 말고 일반 텍스트나 `TODO:`로 둔다.
- `raw/` 파일은 절대 수정하지 않는다.

## 마무리

1. `wiki/index.md`를 실제 파일 수와 맞춘다.
2. `wiki/log.md`에 `## [YYYY-MM-DD] lint | <요약>` 항목을 append한다.
3. 사용자에게 발견 사항, 수정 내역, 남은 리스크를 간결히 보고한다.
