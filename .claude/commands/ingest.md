# /ingest — 원본 인제스트

`raw/`를 스캔해 아직 `wiki/`에 반영되지 않은 원본을 찾아 인제스트한다.
운영 기준은 루트 `CLAUDE.md`의 **Ingest** 작업 정의를 따른다.

## 모드

- 인자 없음 → **인터뷰 모드**: 원본별로 맥락 질문을 한 뒤 반영.
- `$ARGUMENTS`에 `quick`·`바로`·`빠르게` 포함 → **빠른 모드**: 인터뷰 생략, 바로 반영.
- `$ARGUMENTS`에 `dialogue`·`대화`·`현재대화` 포함 → **대화 저장 모드**:
  지금 진행 중인 이 대화를 `raw/dialogues/`에 원본으로 저장한 뒤, 그 파일을 인제스트한다.
  (아래 "대화 저장 모드" 절차를 먼저 수행하고, 이후 일반 절차로 이어진다.)
  `quick`과 함께 쓰면 저장 후 인터뷰 없이 바로 반영한다.

## 대화 저장 모드 (`dialogue`)

`raw/`를 스캔하기 전에, 현재 대화를 클리퍼 원본으로 먼저 만든다.

### D-1. 대화 내용 정리
- 지금까지의 대화에서 **사용자 발화**와 **Claude 응답**을 시간순으로 추린다.
  (도구 호출 로그·시스템 리마인더는 제외하고, 사람이 읽을 본문만 남긴다.)
- 대화의 주제를 한국어 한 줄로 압축해 **제목**을 정한다.
- 대화의 배경·목적을 한 줄로 요약해 **description**(🗂 컨텍스트)을 만든다.

### D-2. 원본 파일 작성
`raw/dialogues/<YYYY-MM-DD> <제목>.md`로 저장한다. 포맷은 옵시디언 웹 클리퍼
템플릿 `etc/clipper-template/clipper-claude.json`과 **동일한 구조**를 따른다:

```markdown
---
source_type: "claude-conversation"
title: "<제목>"
url: ""
clipped: <오늘 날짜 YYYY-MM-DD>
description: "<한 줄 요약>"
status: "raw"
tags:
  - "clippings/claude"
---
> [!info] 원본 대화
> Claude Code 세션 — 공유 URL 없음

## 🗂 컨텍스트
<!-- AI 인제스트 시 이 섹션을 우선 읽는다. 대화의 배경·목적을 한 줄로 요약. -->
<한 줄 요약>

## 💬 대화 본문
**You**

<사용자 발화>

---

**Claude**

<Claude 응답>

---

(… 발화 순서대로 반복 …)

---

## 💭 내 메모
<!-- 왜 저장했나 · 핵심 takeaway · 후속 액션 · 궁금한 점.
     AI 인제스트 시 이 메모를 컨텍스트로 최우선 반영한다. -->

## 🔗 관련 노트
<!-- 연결되는 프로젝트·TIL·문서 링크 -->
```

- 웹 클리퍼와 달리 Claude Code 세션은 공유 URL이 없으므로 `url`은 빈 문자열로 두고,
  `> [!info]` 콜아웃에는 "Claude Code 세션 — 공유 URL 없음"이라 적는다.
- `## 💬 대화 본문`은 기존 `raw/dialogues/`의 다른 파일과 같은 형식
  (`**You**` / `**Claude**` 교대, `---` 구분선)을 그대로 따른다.
- `## 💭 내 메모`·`## 🔗 관련 노트`는 사용자 몫이므로 주석만 남기고 비워 둔다.
  단 인터뷰 모드라면 D-3 인터뷰 답변을 `## 💭 내 메모`에 채워 넣어도 된다.

### D-3. 저장 확인
- 저장한 파일 경로를 사용자에게 알리고, 이어서 일반 인제스트 절차(1단계부터)를 진행한다.
- 방금 만든 이 파일은 당연히 미인제스트 대상이므로 1단계 탐지에서 포함된다.

## 절차

### 1. 미인제스트 원본 탐지
- `raw/`의 모든 파일을 나열한다 (`.gitkeep`·`CLAUDE.md` 제외).
- 이미 인제스트됐는지 확인: `wiki/sources/` 페이지들의 frontmatter `sources:` 목록과
  `wiki/log.md`의 ingest 기록을 대조한다.
- 아직 반영 안 된 파일만 대상으로 삼는다. 대상이 없으면 그 사실을 보고하고 종료.

### 2. 원본 읽기
- 대상 파일을 읽는다. **`raw/`는 절대 수정·삭제하지 않는다.**
- 큰 파일은 부분 읽기. 인라인 이미지는 텍스트를 먼저 읽고 필요 시 이미지를 따로 확인.
- frontmatter의 `source_type`과 본문 `## 💭 내 메모` 섹션을 우선 파악한다.

### 3. 소스 요약 작성
원본별로 정리: **핵심 주장(Key Claims) · 언급된 엔티티(사람·도구·조직) · 다루는 개념(Concepts)**.

### 4. 맥락 인터뷰 — *인터뷰 모드에서만*
원본별로 한 번에 하나씩 질문한다:
1. 이 원본을 왜 캡처/저장했나?
2. 지금 하는 일과 어떻게 연결되나?
3. 이걸로 무엇을 해보고 싶나?

답변은 소스 페이지의 "맥락" 섹션에 우선 반영한다.
빠른 모드에서는 `## 💭 내 메모` 내용을 맥락으로 대신 사용한다.

### 5. 위키 반영
- `wiki/sources/`에 소스 요약 페이지를 생성한다.
- 관련 `wiki/concepts/`·`wiki/flows/` 페이지를 신규 작성하거나 갱신한다.
  - 기존 서술과 모순되면 덮어쓰지 말고 `> ⚠️ 모순:`으로 명시한다.
  - 새 지식은 기존 페이지를 대체하지 않고 새 가지로 추가한다.
- 모든 페이지를 `[[링크형 태그]]`로 상호 연결한다.
- 각 페이지에 frontmatter(`type`, `tags`, `updated`, `sources`)를 넣는다.
- 개념마다 **"왜"**를 포함하고, 가능하면 영균의 학습·실무 맥락과 연결한다.

### 6. 색인·로그 갱신
- `wiki/index.md`: 새 페이지를 카테고리별로 등록하고 통계를 갱신한다.
- `wiki/log.md`: `## [오늘 날짜] ingest | <제목>` 항목을 append한다.

### 7. 요약 보고
생성·갱신한 페이지 목록과 핵심 교차 연결을 사용자에게 간결히 보고한다.
