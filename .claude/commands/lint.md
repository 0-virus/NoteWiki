# /lint — 위키 정리

`wiki/` 전체를 점검해 문제를 찾고 고친다. 운영 기준은 루트 `CLAUDE.md`의 **Lint** 작업을 따른다.
`raw/`는 수정하지 않는다. `wiki/`만 손댄다.

## 1. 점검

`wiki/`의 모든 페이지 + `index.md` + `log.md`를 훑어 다음을 찾는다:

- **모순** — 페이지 간 상충하는 서술.
- **낡은 내용(stale)** — 최신 소스·사실에 의해 갱신이 필요한 서술. `updated`가 오래된 페이지.
- **고아 페이지** — 들어오는 링크(백링크)가 하나도 없는 페이지.
- **깨진 링크** — `[[ ]]`가 존재하지 않는 페이지를 가리킴.
- **누락 개념** — 여러 곳에서 언급되지만 독립 페이지가 없는 중요 개념.
- **누락 상호참조** — 관련 있는데 서로 링크되지 않은 페이지.
- **index 불일치** — `wiki/`의 실제 파일·개수와 `index.md`의 목록·통계가 어긋남.
- **데이터 공백** — 웹 검색으로 메울 수 있는 빈 곳.

## 2. 수정

- 낡은 내용을 업데이트하고 `updated` 날짜를 갱신한다.
  모순은 덮어쓰지 말고 `> ⚠️` 플래그를 남기거나 사실관계를 확인해 정합화한다.
- 누락된 상호참조 링크를 추가하고, 깨진 링크를 고친다.
- 누락된 중요 개념은 페이지를 신설한다 (애매하면 사용자에게 신설을 제안).
- `index.md`를 실제 파일·통계와 일치하도록 정정한다.

## 3. 보고 & 로그

- 발견 사항과 수정 내역을 사용자에게 보고한다.
  추가로 탐구하면 좋을 질문이나 찾아볼 소스도 함께 제안한다.
- `wiki/log.md`에 `## [오늘 날짜] lint | <요약>` 항목을 append한다.
