# /commit — 위키 변경사항 커밋

`wiki/log.md`의 최근 항목을 읽어 커밋 메시지를 생성하고 커밋한다.
`$ARGUMENTS`에 `--dry-run` 포함이면 커밋하지 않고 메시지·스테이지 목록만 출력한다.

## 절차

### 1. 변경 파일 확인

`git status`를 실행해 변경된 파일 목록을 파악한다.
변경이 없으면 그 사실을 보고하고 종료한다.

### 2. 최근 로그 항목 추출

마지막 커밋 이후 추가된 로그 항목을 찾는다.

1. `git log -1 --format="%ci"` 로 마지막 커밋의 날짜(YYYY-MM-DD)를 구한다.
2. `wiki/log.md`에서 `## [YYYY-MM-DD]` 패턴의 항목을 추출하고,
   그 날짜보다 **같거나 이후인 항목**을 모두 수집한다.
3. 항목이 없으면 가장 최근 항목 3개를 사용하고 그 사실을 사용자에게 알린다.

### 3. 커밋 메시지 생성

추출한 로그 항목을 기반으로 아래 규칙에 따라 커밋 메시지를 만든다.

**타입 선택 기준**:
- `feat:` — 새 wiki 페이지·개념 추가, 인제스트, 커맨드 신규 추가
- `docs:` — 기존 페이지 수정·보완, 링크 정리
- `chore:` — `.claude/` 설정·커맨드 수정만 있을 때

**제목** (50자 이내 한국어):
- 항목이 1개: `feat: <항목 제목 요약>`
- 항목이 여럿: 가장 중요한 변경(새 개념 수 기준)을 한 줄로 요약

**본문** (항목이 2개 이상일 때):
```
feat: <주요 변경 한 줄>

- [YYYY-MM-DD] <ingest|query|lint|etc> | <제목>
- [YYYY-MM-DD] <ingest|query|lint|etc> | <제목>
...
```

### 4. 파일 스테이징

다음 경로의 변경 파일을 스테이징한다. **민감 정보 파일(`.env`, 시크릿 등)은 절대 포함하지 않는다.**

| 경로 | 포함 조건 |
|---|---|
| `wiki/` | 항상 포함 |
| `raw/` | 신규 파일 + 대화 보완으로 수정된 파일 |
| `.claude/commands/` | 커맨드 추가·변경이 있을 때 |
| `.claude/settings.json` | 설정 변경이 있을 때 |
| `CLAUDE.md` | 루트 지침 변경이 있을 때 |
| `.claude-memory/` | 메모리 동기화 변경이 있을 때 |

### 5. 커밋 실행

생성한 메시지로 커밋한다.

```
Co-Authored-By: Claude Sonnet 4.6 <noreply@anthropic.com>
```

를 커밋 메시지 끝에 추가한다.

`--dry-run` 이면 스테이지·커밋 없이 메시지와 포함될 파일 목록만 출력하고 종료한다.

### 6. 완료 보고

커밋 해시와 메시지 제목을 사용자에게 보고한다.
