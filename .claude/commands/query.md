# /query — 위키 질의

질문: **$ARGUMENTS**

위 질문에 대해 `wiki/`의 문서를 근거로 답한다. 임베딩/벡터 검색이 아니라
`wiki/index.md` 기반 탐색을 쓴다. 운영 기준은 루트 `CLAUDE.md`의 **Query** 작업을 따른다.

(질문이 비어 있으면 먼저 사용자에게 무엇을 묻고 싶은지 확인한다.)

## 모드

- **기본**: wiki index → 드릴다운 → 답변.
- **`raw/` 파일 지정**: `$ARGUMENTS`에 `raw/`로 시작하는 경로 또는 `raw/` 안에 존재하는
  파일명이 포함되면 **raw 파일 우선 모드**로 전환한다. 나머지 인자를 질문으로 삼는다.
  파일이 `raw/`에 없으면 그 사실을 알리고 기본 모드로 fallback한다.

## raw 파일 우선 모드

wiki index 대신 **지정된 raw 파일을 1차 근거**로 삼아 답한다.

### R-1. raw 파일 읽기
- 지정된 파일을 읽는다. **수정·삭제하지 않는다.**
- frontmatter의 `source_type`과 `## 💭 내 메모` 섹션을 우선 파악해 맥락을 잡는다.
- 질문이 비어 있으면 "이 파일에서 무엇이 궁금하세요?" 하고 사용자에게 확인한다.

### R-2. wiki 연결 보조 탐색
- raw 파일에서 언급된 개념·키워드로 `wiki/index.md`를 조회한다.
- 관련 wiki 페이지가 있으면 읽어서 추가 맥락으로 활용한다.
  (raw 파일이 아직 인제스트 전이라면, wiki에서 유사 개념이 어떻게 정리됐는지 비교할 수 있다.)

### R-3. 종합 답변
- **raw 파일 내용을 1차 출처**로, wiki 페이지를 보조 출처로 답한다.
- raw 파일을 출처로 쓸 때는 `raw/<파일명>` 형태로 표기한다.
- raw 파일과 wiki 내용이 다르면 두 서술을 병기하고 차이를 설명한다.
- 영균의 맥락(학습/실무 단계, 원리 우선)에 맞춰 **"왜"**를 설명한다.

이후 4·5단계(환류·로그)는 기본 모드와 동일하게 진행한다.

## 절차 (기본 모드)

### 1. index 먼저 읽기
`wiki/index.md`를 읽어 질문과 관련된 페이지 후보를 추린다.
이것이 이 위키의 "검색 엔진"이다 — 벡터 DB 대신 색인을 쓴다.

### 2. 드릴다운
후보 페이지(`concepts/`·`flows/`·`sources/`)를 읽는다.
페이지 안의 `[[링크]]`를 따라 필요한 만큼 관련 페이지로 확장한다.
index에서 못 찾으면 `wiki/`를 Grep으로 보조 검색한다.

### 3. 종합 답변
읽은 내용을 종합해 답한다.
- 근거가 된 페이지를 `[[page-name]]` 형태로 **출처 표기**한다.
- 영균의 맥락(학습/실무 단계, 원리 우선)에 맞춰 설명 깊이를 조정하고 **"왜"**를 설명한다.
- 위키에 근거가 없으면 그렇다고 명시한다. 위키 사실과 추측을 구분한다.

### 4. 환류 (file back)
답변이 재사용 가치가 있으면(비교·분석·새로 발견한 연결),
`wiki/`에 새 페이지로 저장할지 제안한다. 동의하면 페이지를 만들고 `index.md`를 갱신한다.
→ 탐색 결과가 채팅으로 사라지지 않고 위키에 누적된다.

### 5. 로그
`wiki/log.md`에 `## [오늘 날짜] query | <질문 요약>` 항목을 append한다.
